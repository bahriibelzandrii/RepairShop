# Система управління майстернею з ремонту комп'ютерів (LabRepair)

Це повноцінний проект для управління процесами у майстерні з ремонту комп'ютерів та електроніки.

## Архітектура
- **Клієнт**: Vite + React 18, React Router v6, Zustand (State), Tailwind CSS, Framer Motion, Axios.
- **Сервер**: Node.js, Express, `better-sqlite3`, JWT авторизація.
- **База Даних**: SQLite (`data/db.sqlite`).

## Інструкція з локального розгортання

1. **Встановлення залежностей**
   В кореневій папці виконайте:
   ```bash
   npm install
   npm install --workspace=server
   npm install --workspace=client
   ```

2. **Ініціалізація Бази Даних**
   ```bash
   node server/init-db.js
   ```
   *Це створить БД в папці `/data` і додасть тестового адміністратора (`admin@repair.shop` / `admin123`).*

3. **Запуск проекту**
   ```bash
   npm run dev
   ```
   *Клієнт буде доступний за адресою `http://localhost:3000`, сервер - `http://localhost:5000`.*

## Деплой на Vercel

Цей проект налаштовано для монорепозиторного деплою на Vercel (Front + Back разом).

1. Створіть новий проект на Vercel.
2. Приєднайте цей репозиторій.
3. У налаштуваннях Vercel:
   - Root Directory: `.` (корінь)
   - Framework Preset: `Vite`
   - Build Command: `npm run build:client`
   - Output Directory: `client/dist`
4. Змінні оточення (Environment Variables) у Vercel:
   - `JWT_SECRET` = ваш_секретний_ключ
   - `JWT_REFRESH_SECRET` = ваш_секретний_ключ_оновлення
   - `NODE_ENV` = `production`

**УВАГА щодо Vercel Serverless та SQLite:**
Vercel використовує ephemeral (тимчасову) файлову систему для Serverless функцій. Файли, що створюються/змінюються (наприклад, `db.sqlite`), зникнуть після завершення виконання функції (cold start).
- *Рішення для лабораторного/тестового деплою:* База даних читатиметься як Read-Only або скидатиметься. 
- *Рішення для продакшену:* Потрібно замінити `better-sqlite3` на Turso (LibSQL), Supabase, або Neon PostgreSQL. У рамках цього проекту збережена вимога використовувати SQLite, тому дані зберігатимуться до рестарту serverless функції.

## Тестування системи
1. Зайдіть під **admin@repair.shop** / **admin123**.
2. Перейдіть до **Клієнтів** та додайте нового клієнта.
3. Перейдіть до **Пристроїв** та додайте пристрій для клієнта (спробуйте завантажити фото).
4. Перейдіть до **Замовлень**, створіть замовлення.
5. Зайдіть в деталі замовлення (через таблицю), додайте послуги (наприклад, Діагностика) та деталі.
6. Перейдіть до **Фінансів** (або всередині замовлення) та додайте оплату.
7. Роздрукуйте квитанцію (кнопка PDF).
8. Спробуйте роути для різних ролей: створіть працівника або клієнта і увійдіть під ними.
